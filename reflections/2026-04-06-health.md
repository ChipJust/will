## What Was Built

### Research Ingest Tooling (`tools/extract/`, `tools/ingest.py`)
Complete pipeline for converting research papers and video transcripts into versioned
markdown in `research/refs/`. Architecture:

- **5 extractor tools** — each does exactly one thing, writes markdown to stdout:
  - `from_pymupdf.py` — PDF primary (pymupdf4llm)
  - `from_pdfminer.py` — PDF fallback (different engine, better for two-column)
  - `from_docx.py` — DOCX via pypandoc
  - `from_html.py` — URL or HTML via trafilatura
  - `from_vtt.py` — VTT transcripts; handles YouTube progressive caption format
- **`quality.py`** — markdown → JSON quality score (word count, valid-word ratio,
  median line length, noise ratio → composite 0–100, verdict ok/marginal/bad)
- **`tools/ingest.py`** — thin CLI orchestrator; detects format, runs extractor,
  quality-checks, saves to `research/refs/<slug>.md` with YAML frontmatter header

### `ingest-paper` Skill (global plugin)
Decision tree skill in `~/.claude/plugins/.../health-ingest/`:
- Directs agent through the fallback chain (pymupdf → pdfminer → docx → html → marker placeholder)
- Reference docs: quality thresholds, format detection (DOI, PMC, arXiv, bioRxiv), fallback logic

### 54 Unit Tests
`tests/test_extract.py` — covers quality scoring math, VTT parsing edge cases,
slugify, title extraction, format detection.

### 24 Research Papers Ingested
All papers cited in existing health research files. 6 full-text (PMC), 18 abstracts
(PubMed). Paywalled papers (NEJM, Nature, Neurology) saved as abstracts; can be
upgraded when PDFs are available.

### Folder Organization
- `research/refs/` — ingested external references
- `research/` — our own notes and analysis (unchanged)

## What Worked Well

- **Clean interface design** (one file, one purpose, stdout) made each piece easy
  to test and easy to wire together. The skill could direct the orchestrator cleanly.
- **Quality scoring** gave a concrete signal for fallback decisions without requiring
  the agent to eyeball the text.
- **Progressive dedup for YouTube VTT** — catching this before the first conversion
  saved the transcript from being 3× the right length with all the carry-forward lines.
- **Parallel ingest of 24 papers** via subagent — efficient once the tools were solid.
- **Encoding fix** surfaced naturally (papers with Greek/math symbols crashed on
  Windows cp1252). Fixed in one pass across both affected files.

## What Could Be Improved

- **Test fixtures** had wrong expectations in 4 cases on first run (CLEAN_PROSE
  word count, garbled text detection, slugify special chars). Tests should be run
  mentally against the actual algorithm before writing assertions.
- **`ingest.py` exits 1 on bad quality** which prevents the agent from seeing the
  output. Should probably save anyway and exit 2 as a signal only (this was the
  intent; verify the behavior is consistent).
- **Abstract-only ingests** get "marginal" quality scores that are technically fine
  but misleading — a 18-word file is not "marginal", it's nearly empty. Consider
  adding a minimum word count hard gate (e.g. < 50 words → always "bad" regardless
  of other scores).

## Decisions Made

- `research/refs/` subdirectory pattern — separates ingested references from
  authored notes. Simple, clear, no naming convention needed.
- `--slug` from title/metadata rather than always requiring user input — reduces
  friction for the common case.
- Marker placeholder rather than dependency — right call; 500MB ML models are not
  worth it until we actually hit a paper that fails both PDF engines.
